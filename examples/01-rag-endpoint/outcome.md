For this example, the app was designed first and Layer 0 was run afterward to show what the output looks like. In practice, Layer 0 runs once at onboarding, before any stories are worked.

## What Was Built

A POST `/api/v1/questions/ask` endpoint implementing retrieval-augmented generation:
- `QuestionController` — HTTP handler, delegates to service
- `QuestionService` — RAG orchestration, prompt construction, answer generation
- `AskRequest` / `AskResponse` DTOs
- `VectorStoreException` / `ModelException` domain exceptions
- `GlobalExceptionHandler` mapping exceptions to HTTP 503
- Prompt template at `resources/prompts/question.st`
- Unit tests for service (Mockito), controller tests (MockMvc)

## What the Design Conversation Prevented

| Problem | Level caught | Cost if discovered at Level 5 |
|---------|-------------|-------------------------------|
| Unnecessary `PromptBuilder` abstraction | Level 2 | 1 class + 1 test + refactor request |
| Empty-result behavior undefined | Level 2 | Ambiguous implementation + back-and-forth |
| Service interface vs. class convention | Level 4 | Structural refactor of generated code |

The service interface correction at Level 4 was caught because `layer-0-design-principles.md` explicitly states: "Do not create service interfaces — `@Service` classes directly." Without that design constraint in context, the AI would not have been wrong to propose an interface — it's a common Java pattern. The constraint is what made the deviation visible before implementation.

## What the Layer 0 Files Added

The two Layer 0 output files changed the quality of the design conversation in one specific way: they replaced general corrections with specific references.

Without them: "Don't add a PromptBuilder — it's unnecessary abstraction."
With them: "PromptBuilder is not in the ARCHITECTURE.md component list. Remove it."

The design conversation moves faster when corrections reference shared documents rather than restating reasoning from scratch.

## Code Generated at Level 5

The Level 5 implementation required zero structural corrections. Minor adjustments:
- Logging statement format aligned to project convention (parameter ordering)
- Javadoc on one method was too brief — expanded
