## What Was Built

A standalone Angular 17 component for searching users:
- `UserSearchComponent` — debounced `FormControl`, explicit loading/error/filteredUsers state
- `UserService` — single `getUsers()` method, HTTP only
- `User` model interface
- Template with four mutually exclusive state blocks (loading, error, empty, results)
- 8 unit tests: load, filter by name/username/email, empty query, no match, error, retry

## What the Design Conversation Prevented

| Problem | Level caught | Cost if discovered at Level 5 |
|---------|-------------|-------------------------------|
| Unnecessary `SearchResultsComponent` | Level 2 | New file + spec file + `@Input()` binding + refactor |
| async pipe / Observable state properties | Level 4 | Full component rewrite — loading/error state incompatible with async pipe approach |

The async pipe correction at Level 4 is the more important catch. Copilot defaulted to
`Observable<T>` properties and async pipe — a standard Angular pattern, not a mistake in
isolation. What made it wrong here was the project constraint documented in
`layer-0-design-principles.md`: "Do not use async pipe when loading/error state is needed."

Without that constraint in the loaded context, the correction would not have surfaced at Level 4.
It would have arrived after Level 5 generated a fully wired async-pipe implementation that
required a complete rewrite to add proper loading and error handling.

## What the Layer 0 Files Added

The Layer 0 files contained the "Do not use async pipe" constraint under Design Constraints.
When Copilot proposed Observable properties at Level 4, the human referenced the document
rather than re-explaining the rationale. The conversation took one exchange to correct and move on.

Without the Layer 0 files: "Don't use async pipe here — we need loading and error state."
With the Layer 0 files: "See layer-0-design-principles.md Design Constraints. Use explicit properties."

The document shortened the correction from an explanation to a reference.

## Code Generated at Level 5

The Level 5 implementation required zero structural corrections. Minor adjustments:
- `aria-label` attribute added to the search input (missed in contracts)
- SCSS nesting depth reduced by one level for readability
