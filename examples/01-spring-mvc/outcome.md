## What Was Built

A four-endpoint Spring MVC REST API for an in-memory product catalog:
- `ProductController` — HTTP contract, delegates immediately to service
- `ProductService` — business logic, `ConcurrentHashMap` store
- `Product` / `CreateProductRequest` — validated records
- `ProductNotFoundException` — domain exception
- `GlobalExceptionHandler` — maps exceptions to HTTP 404 / 400
- `ErrorResponse` — standardized error body
- 8 service tests (plain JUnit), 9 controller tests (`@WebMvcTest`)

## What the Design Conversation Prevented

| Problem | Level caught | Cost if discovered at Level 5 |
|---------|-------------|-------------------------------|
| Unnecessary `ProductRepository` interface + `InMemoryProductRepository` | Level 2 | 2 files + Spring bean + injection wiring + refactor |
| `Long` IDs instead of `UUID` | Level 4 | Every method signature, every test, every URL example — full rewrite |

The `Long`-to-UUID catch at Level 4 is the more consequential correction. Copilot defaulted
to `Long` — a legitimate Java convention, not a mistake in isolation. The constraint
"UUID only" documented in `layer-1-base-instructions.md` and `layer-0-design-principles.md`
made it visible before a single line of code existed. Without that constraint in context,
a UUID refactor would have arrived as a structural correction across the entire codebase.

## What the Layer 0 Files Added

The Layer 0 files contained two Design Constraints that drove both corrections:

- `layer-0-architecture.md`: "Do not add a repository layer — service owns the store directly"
- `layer-1-base-instructions.md`: "UUID for all product identifiers — never Long or int"

Both corrections at Level 2 and Level 4 took one exchange each. The human referenced
the document — no explanation required. Without the Layer 0 files, both conversations
would have required explanation from scratch, and the Level 4 correction might not have
surfaced until the code was already written.

## Code Generated at Level 5

The Level 5 implementation required zero structural corrections. Minor adjustments:
- Javadoc on `SpringMvcDemoApplication.main()` added (missed in contracts)
- `listAll` test added a multi-item case to confirm the store grows correctly
